========== SLIDE 2 ==========

1. Introduction
Kabaddi is a high-intensity contact sport that requires explosive movements such as toe-touch, scorpion kick, hand touch, dubki, bonus step, and jump raid. Mastery of these skills demands precise timing, balance, and body coordination.

Players—especially beginners—often struggle to correctly learn these techniques because visual demonstrations are brief, coaches may not always be available, and players cannot see their own mistakes in real time.

This project introduces an AR-based Kabaddi Ghost Trainer, where a virtual avatar demonstrates reference moves and the system evaluates how closely the player replicates them using pose estimation, sequence comparison, and feedback generation.

The goal is to help Kabaddi learners practice independently, receive real-time correction cues, and improve movements with clear visual and data-driven guidance.


========== SLIDE 3 ==========

2. Introduction (Kabaddi skills)
Toe Touch
A quick forward extension of the leg to tap the defender while maintaining balance.
Scorpion Kick
A backward high kick performed when facing away from the defender.

A leap over defenders to avoid blocks while maintaining raid rhythm.



Hand Touch / Running Hand Touch
Fast hand extension during forward motion to tag defenders.

Dubki
A low-body dive used to slip under a defender’s block.
Bonus Step
A long stride into the bonus line using body stretch and hip flexibility.
Ankle Hold Pose

Thigh Hold Pose

Chain Tackle Formation Pose
Coordinated multi-player defensive posture to tackle a rider
Running Raid Pose
Continuous forward locomotion Used before hand touch or toe touch


========== SLIDE 4 ==========

2. Problem Statement
Problem:
Kabaddi learners lack continuous visual guidance, real-time correction, and objective scoring, making it difficult to accurately replicate advanced movements.

Solutions Provided By the Proposed System:
AR Playback of Reference Moves
Players can view a 3D/AR “ghost” demonstrating the correct technique, allowing them to mimic the move with proper angle and timing.
Automated Movement Analysis
The user's captured movement is processed using pose estimation and compared with expert reference sequences.
Error Identification
Joint-wise mistakes, timing mismatches, and posture deviations are detected using similarity metrics and threshold-based classifiers.
LLM-Generated Coaching Feedback
The system produces simple, actionable instructions (“Raise your right elbow higher”, “Lean less forward”, etc.) to guide improvement.

6A (i) – Algorithm Details
Algorithms Used
YOLOv8 – Object Detection
Role: Detect all human instances in each frame.
Result: B = { (x1, y1, x2, y2, confidence, class=person) }
ByteTrack – Multi-object Tracking
Role: Assign persistent IDs to detected persons across frames.
Result: Track ID → Sequence of bounding boxes over time
Motion-Based Raider Selection (Heuristic) “Cumulative Displacement Heuristic”
Formula: 
		motion_i = Σ || position_t − position_(t−1) ||
		raider = argmax(motion_i)

CNN-based pose estimation model

Pose Representation

========== SLIDE 21 ==========

6A(iv) Algorithms Used
Type:
	Deterministic Signal-Processing Pipeline (5 Steps)
Step 1: Temporal Interpolation
Algorithm: (Rule Based)	Linear Interpolation Across Time
Rule
 Interpolates only if at least two valid frames exist for a joint

Step 2: Pelvis Centering (Translation Normalization)
Algorithm:	Pelvis-Based Centering
Formula:
	pelvis(t) = (left_hip + right_hip) / 2
	joint_centered = joint − pelvis(t)


========== SLIDE 22 ==========

6A(iv) Algorithms Used (cont)
Step 3: Scale Normalization
Algorithm	Torso-Length Normalization
Formula
	torso_length = || shoulder_center − hip_center ||
	joint_normalized = joint_centered / torso_length
Step 4: Outlier Suppression
Algorithm	Z-Score Velocity-Based Outlier Detection
Logic
velocity(t) = || pose(t) − pose(t−1) ||
z = (velocity − μ) / σ

If |z| > 3.0 → replace pose(t) with pose(t−1)

========== SLIDE 28 ==========

6B (ii) – Algorithm Details
Algorithm Type
Dynamic Programming-Based Temporal Alignment

Step 1: Cost Matrix Construction
Distance Metric
	For expert frame i and user frame j:
		d(i, j) = || expert_pelvis[i] − user_pelvis[j] ||

Step 2: Optimal Path Backtracking
Backtracking Rule
Start from:
(i, j) = (T_expert, T_user)

At each step, move to the minimum of:
D[i−1, j−1], D[i−1, j], D[i, j−1]

Termination condition :
	(i == 0 AND j == 0)

========== SLIDE 35 ==========

6C (i) Algorithm Details
Algorithm Type
	Deterministic Euclidean Distance Computation
Frame-wise Joint Error Formula
For frame t and joint j:
error(t, j) = || expert(t, j) − user(t, j) ||₂
error(t, j) = sqrt( (x_e − x_u)² + (y_e − y_u)² )
Where:
	(x_e, y_e) → expert joint coordinates
	(x_u, y_u) → user joint coordinates

========== SLIDE 47 ==========

6D (i) Scoring Algorithms
Algorithm Type
	Deterministic Score Aggregation

1. Structural Similarity Score
	Extract mean joint errors: mean_errors = { mean_error(j) for all 17 joints }
	Compute overall mean joint error: mean_joint_error = nanmean(mean_errors)
	Normalize using fixed threshold: structural_similarity = (1 − mean_joint_error / MAX_ERROR_THRESHOLD) × 100
Where:	MAX_ERROR_THRESHOLD = 1.5 Clamp result to [0, 100].


2. Temporal Similarity Score
Extract aligned frame count: T_aligned
Compare with baseline:
BASELINE_FRAMES = 115
Compute deviation:
frame_deviation = |T_aligned − BASELINE_FRAMES|
Normalize deviation:
temporal_quality =
1 − (frame_deviation / (BASELINE_FRAMES × 0.5))
Scale to score range:
temporal_similarity = 70 + (temporal_quality × 30)
